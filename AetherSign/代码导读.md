# 代码导读

## 一、从哪里开始看

如果你只想快速跑起来，阅读顺序建议是：

1. `configs/aethersign_s2g_v0.yaml`
2. `scripts/train_s2g.py`
3. `mska_v0/engine.py`
4. `mska_v0/data.py`
5. `mska_v0/model.py`
6. `mska_v0/recognition.py`
7. `mska_v0/visual_head.py`

## 二、每个文件负责什么

### 1. `configs/aethersign_s2g_v0.yaml`

这是整个实验的主配置文件，里面定义了：

- 数据路径
- 关键点格式
- 四个流的索引划分
- 训练超参数
- 推理 beam size
- 输出目录

如果你们后续改关键点定义，最优先改这里。

### 2. `scripts/build_gloss_vocab.py`

负责读取 `gloss_map.txt`，生成 `gloss2ids.pkl`。

这个文件是训练前必跑的一步，因为 CTC 和 tokenizer 都要依赖它。

### 3. `scripts/train_s2g.py`

训练入口。核心流程是：

1. 读取配置
2. 构造 tokenizer
3. 构造 train/dev/test dataloader
4. 构造模型、优化器、调度器
5. 训练
6. 每轮在 dev 上评估
7. 保存 `last` 和 `best` checkpoint

### 4. `scripts/infer_s2g.py`

推理入口。支持两种模式：

- `--split dev/test`：整套评估
- `--npy path.npy`：单样本推理

### 5. `mska_v0/data.py`

这是这次复现里最关键的适配文件之一。

它负责：

- 读取 `train/dev/test.csv`
- 懒加载 `.npy`
- 检查 `C=3`、`V=65`、`T` 是否与标注一致
- 做时序采样与 padding
- 做坐标预处理
- 生成模型输入 batch

这里实现了你们这套数据格式和原始 MSKA 模型之间的桥梁。

### 6. `mska_v0/model.py`

最外层模型封装。当前只保留 `S2G`。

真正的识别网络在 `recognition.py`。

### 7. `mska_v0/recognition.py`

这是最核心的网络实现。

主要包含：

- `STAttentionBlock`
- `DSTA`
- `Recognition`

其中：

- `DSTA` 负责四流关键点注意力主干
- `Recognition` 负责四个 head、CTC loss、自蒸馏 loss 和 CTC beam search 解码

和原仓库相比，最大的改动是：

- 节点数不再写死
- `.cuda()` 改成设备无关

### 8. `mska_v0/visual_head.py`

每个流后面的 head 网络。

它做的是：

- 线性映射
- 归一化
- 位置编码
- 前馈时序模块
- 最终分类

### 9. `mska_v0/tokenizer.py`

负责 gloss 和 id 的互转，以及生成 CTC 训练需要的 label tensor。

### 10. `mska_v0/engine.py`

这里是脚本共用逻辑层，主要负责：

- 配置解析
- 路径解析
- dataloader 构建
- 模型构建
- checkpoint 加载
- 评估与 WER 汇总
- 单文件推理

### 11. `mska_v0/metrics.py`

只保留了本任务需要的 WER 计算逻辑。

### 12. `mska_v0/optimizer.py`

负责 Adam / AdamW / SGD 与 cosine scheduler。

### 13. `mska_v0/utils.py`

放了一些基础工具，包括：

- `MetricLogger`
- `SmoothedValue`
- `MaskedNorm`
- `PositionalEncoding`
- `PositionwiseFeedForward`

## 三、一次训练的数据流

训练时的数据流大致如下：

1. `train_s2g.py` 读取 YAML
2. `engine.py` 构建 dataloader
3. `data.py` 读入一批 `.npy`
4. `data.py` 做采样、padding、坐标变换
5. batch 喂给 `model.py`
6. `model.py` 调用 `recognition.py`
7. `recognition.py` 进入四流 DSTA 主干
8. 四个流分别进入各自 `visual head`
9. 计算四个 CTC loss
10. 再叠加自蒸馏 loss
11. 返回 `total_loss`
12. `train_s2g.py` 反向传播并更新权重

## 四、如果后面要继续改，优先改哪里

### 场景 1：关键点定义变了

优先改：

- `configs/aethersign_s2g_v0.yaml`

主要改这些字段：

- `data.num_keypoints`
- `model.RecognitionNetwork.DSTA-Net.left`
- `model.RecognitionNetwork.DSTA-Net.right`
- `model.RecognitionNetwork.DSTA-Net.face`
- `model.RecognitionNetwork.DSTA-Net.body`

### 场景 2：坐标不是归一化图像坐标

优先改：

- `configs/aethersign_s2g_v0.yaml`
- `mska_v0/data.py`

### 场景 3：以后要做开发板轻量化 v1

优先看：

- `mska_v0/recognition.py`
- `mska_v0/visual_head.py`

因为大部分算子替换都会集中在这两个文件。
